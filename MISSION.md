# Mission: Learn nginx from first principles to production

## Why
The user wants a comprehensive understanding of nginx across multiple use cases:
- Deploy and host web applications
- Perform DevOps and sysadmin tasks
- Understand infrastructure fundamentals
- Optimize performance and reliability
- Set up local development environments

## Current State
- **Experience**: Complete beginner with nginx
- **Immediate goal**: Understand what nginx is and how it works, then serve a static site
- **Long-term trajectory**: From basics → static serving → reverse proxying → SSL/TLS → performance tuning

## Zone of Proximal Development
Start with conceptual foundations and hands-on basics. The user needs to:
1. Understand what nginx is and why it exists
2. Get it running locally and serve static files
3. Then progressively add complexity: reverse proxying, SSL, advanced configs

## Notes
- Teaching should span knowledge (what is nginx) and skills (can you configure it)
- Lessons should be small, self-contained, and directly tied to practical wins
- User wants to understand deeply, not just copy-paste configs
