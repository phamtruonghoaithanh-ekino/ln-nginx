nginx Glossary & Quick Reference

Glossary

Master process
The root process that reads config, binds sockets, and spawns/manages workers. Runs as root.
Worker process
Single-threaded process that handles connections via an event loop. Does the actual request processing.
Event loop
Non-blocking I/O mechanism (epoll on Linux, kqueue on BSD/macOS) that lets one worker handle thousands of connections concurrently.
Context
A block directive that contains other directives. Main contexts: main, events, http, server, location, upstream.
Directive
A configuration instruction. Simple: name params;. Block: name { ... }.
Location
A context inside server that matches request URIs by prefix or regex.
Root
Directive that sets the filesystem path for serving files. Appends the full URI to the path.
Alias
Like root, but replaces the matched location prefix instead of appending the full URI.
Upstream
A named group of backend servers used for reverse proxying and load balancing.
Reverse proxy
nginx receives a request, forwards it to a backend server (app server), and returns the response to the client.
proxy_pass
Directive that sends requests to a proxied server (URL or upstream group).
Sendfile
OS-level optimization that copies data from disk directly to a socket (bypassing userspace). Used for serving static files.
C10K problem
The challenge of handling 10,000 simultaneous connections. nginx was designed specifically to solve this.
Server block
nginx's equivalent of Apache's "virtual host". Defines a site listening on a specific port/hostname.

Location Matching Rules

SyntaxTypePrecedence
location = /pathExact matchHighest
location ^~ /pathPrefix, no regex checkHigh
location ~ patternRegex (case-sensitive)Medium
location ~* patternRegex (case-insensitive)Medium
location /pathPrefix (standard)Lowest (longest match wins)

Priority order: exact > prefix with ^~ > regex (~ / ~*) > plain prefix (then longest wins).
